## Repository and history review

The tracked tree and all reachable Git commits were checked for common private-key markers, cloud-token prefixes, GitHub personal-access-token prefixes, App Store Connect key filenames, provisioning profiles, and other sensitive signing filenames. No credential-shaped match or tracked signing material was found.

The current tree contains no machine-specific owner home path. Preview and test fixtures use explicit synthetic accounts such as `/Users/preview` and `/Users/example`; a test also asserts that copied AI-session output contains no `/Users/` path.

Published commits retain the author's earlier Gmail address. This is intentional: history will not be rewritten solely to replace an author email. Future repository-local author configuration uses `Rocco Dâ€™Affuso <265598188+roccodaffuso@users.noreply.github.com>`.

GitHub secret scanning, push protection, and private vulnerability reporting are release controls in addition to this local pattern audit. A pattern scan cannot prove the absence of every possible secret, so any later discovery must trigger immediate credential revocation before evaluating a history rewrite.

## Dependencies and asset provenance

`Package.swift` declares no third-party package dependencies. Corewise uses Swift, SwiftUI, AppKit/macOS frameworks, and repository-owned source only. There is no third-party notices bundle required for the current app.

The application icon is repository-owned and reproducibly generated by `script/generate_app_icon.swift`. `Sources/Corewise/Resources/Corewise.icns` is the packaged icon and `docs/assets/corewise-app-icon.png` is its public preview. The design-audit screenshot is produced from Corewise itself and contains no third-party artwork.

Contributor Covenant text is adapted from version 2.1 with attribution in `CODE_OF_CONDUCT.md`.

## Distribution boundary

Release signing certificates, private keys, App Store Connect `.p8` files, notary credentials, submission logs containing account metadata, and Keychain exports must remain outside Git and release assets.

Every public app bundle must contain:

- `LICENSE.txt`, the full MPL-2.0 text;
- `SourceCode.txt`, identifying the canonical corresponding-source repository.

The notarized `0.1.0 (1)` DMG predates the permanent bundle identifier and bundled MPL/source notice. The notarized `0.1.0 (2)` candidate was superseded before publication by a keyboard-focus fix. Public beta must start with a freshly signed and notarized build `3` or later.
